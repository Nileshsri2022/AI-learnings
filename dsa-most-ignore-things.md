### 8. HANDLING TLE & DEBUGGING (Practical Skills)

```
WHEN YOUR SOLUTION GETS TLE (Time Limit Exceeded):
в”Ӯ
в”ңв”Җв”Җ CHECKLIST:
в”Ӯ   в”ңв”Җв”Җ 1. Read constraints again вҶ’ determine required complexity
в”Ӯ   в”ңв”Җв”Җ 2. Are you using correct algorithm?
в”Ӯ   в”Ӯ   в””в”Җв”Җ O(nВІ) when O(n log n) is needed?
в”Ӯ   в”ңв”Җв”Җ 3. Are you doing unnecessary work inside loops?
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ String concatenation in loop (use StringBuilder)
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ Sorting inside loop
в”Ӯ   в”Ӯ   в””в”Җв”Җ Map operations that could be precomputed
в”Ӯ   в”ңв”Җв”Җ 4. Is your recursion without memoization?
в”Ӯ   в”Ӯ   в””в”Җв”Җ Add dp[] array to cache results
в”Ӯ   в”ңв”Җв”Җ 5. Can you use a different data structure?
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ Array вҶ’ HashMap (O(n) вҶ’ O(1) lookup)
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ List вҶ’ Set (O(n) вҶ’ O(1) contains)
в”Ӯ   в”Ӯ   в””в”Җв”Җ Sorting + Binary Search instead of linear search
в”Ӯ   в””в”Җв”Җ 6. Language-specific optimizations:
в”Ӯ       в”ңв”Җв”Җ C++: Use '\n' instead of endl (flushes buffer)
в”Ӯ       в”ңв”Җв”Җ C++: Use scanf/printf instead of cin/cout
в”Ӯ       в”ңв”Җв”Җ Java: Use BufferedReader instead of Scanner
в”Ӯ       в””в”Җв”Җ Python: Use sys.stdin for faster input
в”Ӯ
в”ңв”Җв”Җ COMMON OPTIMIZATION PATTERNS:
в”Ӯ   в”ңв”Җв”Җ O(nВІ) вҶ’ O(n log n): Sort + Binary Search / Two Pointers
в”Ӯ   в”ңв”Җв”Җ O(nВІ) вҶ’ O(n): HashMap / Sliding Window / Prefix Sum
в”Ӯ   в”ңв”Җв”Җ O(2вҒҝ) вҶ’ O(nВІ) or O(n*target): DP (memoization)
в”Ӯ   в””в”Җв”Җ O(n*m) вҶ’ O(n+m): Two Pointer on sorted data
в”Ӯ
DEBUGGING RECURSIVE CODE:
в”Ӯ
в”ңв”Җв”Җ TECHNIQUES:
в”Ӯ   в”ңв”Җв”Җ 1. Print the function call with parameters at entry
в”Ӯ   в”ңв”Җв”Җ 2. Print the return value before returning
в”Ӯ   в”ңв”Җв”Җ 3. Draw the recursion tree on paper (ESSENTIAL)
в”Ӯ   в”ңв”Җв”Җ 4. Verify base cases with smallest inputs
в”Ӯ   в”ңв”Җв”Җ 5. Check: Are you passing by value or reference?
в”Ӯ   в””в”Җв”Җ 6. Check: Is memoization key correct and unique?
в”Ӯ
в””в”Җв”Җ COMMON BUGS:
    в”ңв”Җв”Җ Off-by-one errors in loops (< vs <=)
    в”ңв”Җв”Җ Integer overflow (use long long in C++)
    в”ңв”Җв”Җ Not handling empty input / null pointer
    в”ңв”Җв”Җ Modifying collection while iterating
    в”ңв”Җв”Җ Wrong base case in recursion
    в”ңв”Җв”Җ Forgetting to mark visited in graph problems
    в””в”Җв”Җ Using global variables that don't reset between test cases
```

### 12. MENTAL HEALTH & CONSISTENCY

```
NOBODY TALKS ABOUT THIS, BUT IT'S REAL:

BURNOUT PREVENTION:
в”ңв”Җв”Җ Take 1 full day OFF per week (no DSA, no coding)
в”ңв”Җв”Җ Sleep 7-8 hours (your brain consolidates learning during sleep)
в”ңв”Җв”Җ Exercise / walk daily (improves cognitive function)
в”ңв”Җв”Җ Don't compare with others ("He solved 500 problems!")
в”Ӯ   в””в”Җв”Җ YOUR 300 quality problems > someone's 500 copy-pasted solutions
в”ңв”Җв”Җ It's OK to have bad days (some days you'll solve nothing)
в””в”Җв”Җ Progress is NOT linear вҖ” you'll have plateaus

WHEN YOU'RE STUCK ON A PROBLEM:
в”ңв”Җв”Җ Spent 30-40 min? вҶ’ Read just the HINT, not full solution
в”ңв”Җв”Җ Spent 60 min? вҶ’ Watch the solution video
в”ңв”Җв”Җ DON'T just copy code вҖ” understand WHY it works
в”ңв”Җв”Җ Come back to it in 3 days and solve it fresh
в””в”Җв”Җ Keep a "retry list" of problems you couldn't solve

CONSISTENCY > INTENSITY:
в”ңв”Җв”Җ 2 hours DAILY for 6 months >>> 12 hours for 2 weeks then quit
в”ңв”Җв”Җ Use habit stacking: "After morning coffee, I do DSA for 1 hour"
в”ңв”Җв”Җ Track streaks (LeetCode has streak counter)
в”ңв”Җв”Җ Find an accountability partner (friend doing same prep)
в””в”Җв”Җ Join communities:
    в”ңв”Җв”Җ r/leetcode (Reddit)
    в”ңв”Җв”Җ Striver's Discord/Telegram
    в”ңв”Җв”Җ NeetCode Discord
    в””в”Җв”Җ Your college coding club

HANDLING REJECTION:
в”ңв”Җв”Җ You WILL get rejected вҖ” sometimes after clearing 4/5 rounds
в”ңв”Җв”Җ Each rejection teaches you something
в”ңв”Җв”Җ Keep applying while preparing (don't wait to be "ready")
в”ңв”Җв”Җ Apply to 50+ companies, not just 3-4
в”ңв”Җв”Җ Off-campus > On-campus opportunities in many cases
в””в”Җв”Җ First job doesn't define your career
```
