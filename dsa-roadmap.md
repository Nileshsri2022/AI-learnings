```

в”ңв”Җв”Җ O(nВІ) вҶ’ Quadratic   : Nested loops

в”Ӯ
в”ңв”Җв”Җ Space Complexity:
в”Ӯ   в”ңв”Җв”Җ Auxiliary space vs Total space
в”Ӯ   в”ңв”Җв”Җ Recursion stack space
в”Ӯ   в””в”Җв”Җ In-place vs extra space algorithms
в”Ӯ
в”ңв”Җв”Җ HOW TO ANALYZE:
в”Ӯ   в”ңв”Җв”Җ Single loop вҶ’ O(n)
в”Ӯ   в”ңв”Җв”Җ Nested loop вҶ’ O(nВІ)
в”Ӯ   в”ңв”Җв”Җ Loop with i = i*2 вҶ’ O(log n)
в”Ӯ   в”ңв”Җв”Җ Two separate loops вҶ’ O(n) + O(m)
в”Ӯ   в””в”Җв”Җ Recursion вҶ’ Draw recursion tree вҶ’ count nodes
в”Ӯ
в””в”Җв”Җ PRACTICE:
    в””в”Җв”Җ Analyze complexity of 15-20 code snippets
```

**Test yourself:** Can you tell the complexity of this?
```cpp
for(int i = 0; i < n; i++) {          // O(n)
    for(int j = i; j < n; j++) {      // O(n) вҖ” but starts from i
        cout << i << j;               // O(1)
    }
}
// Answer: O(nВІ) вҖ” n + (n-1) + (n-2) + ... + 1 = n(n+1)/2
```

### 6A. 1D DP (Week 18-19)

```
LEARN THE FRAMEWORK:
в”Ӯ
в”ңв”Җв”Җ PROBLEM: Climbing Stairs (LC 70)
в”Ӯ
в”Ӯ   STEP 1 вҖ” Recursion:
в”Ӯ   int solve(n):
в”Ӯ       if n <= 1: return 1
в”Ӯ       return solve(n-1) + solve(n-2)
в”Ӯ   // Time: O(2вҒҝ) вҖ” TOO SLOW
в”Ӯ
в”Ӯ   STEP 2 вҖ” Memoization (Top-Down):
в”Ӯ   int solve(n, dp[]):
в”Ӯ       if n <= 1: return 1
в”Ӯ       if dp[n] != -1: return dp[n]
в”Ӯ       return dp[n] = solve(n-1, dp) + solve(n-2, dp)
в”Ӯ   // Time: O(n), Space: O(n)
в”Ӯ
в”Ӯ   STEP 3 вҖ” Tabulation (Bottom-Up):
в”Ӯ   dp[0] = 1, dp[1] = 1
в”Ӯ   for i = 2 to n:
в”Ӯ       dp[i] = dp[i-1] + dp[i-2]
в”Ӯ   return dp[n]
в”Ӯ   // Time: O(n), Space: O(n)
в”Ӯ
в”Ӯ   STEP 4 вҖ” Space Optimization:
в”Ӯ   prev2 = 1, prev1 = 1
в”Ӯ   for i = 2 to n:
в”Ӯ       curr = prev1 + prev2
в”Ӯ       prev2 = prev1
в”Ӯ       prev1 = curr
в”Ӯ   return prev1
в”Ӯ   // Time: O(n), Space: O(1) вң…
в”Ӯ

### 7A. Greedy Algorithms (Week 24)

```
CONCEPT:
в”ңв”Җв”Җ Make locally optimal choice at each step
в”ңв”Җв”Җ Hope it leads to globally optimal solution
в”ңв”Җв”Җ PROOF: Usually by contradiction or exchange argument
в”Ӯ
в”ңв”Җв”Җ KEY DIFFERENCE FROM DP:
в”Ӯ   DP: Try ALL choices вҶ’ pick best
в”Ӯ   Greedy: Pick BEST choice now вҶ’ never look back
в”Ӯ
в”ңв”Җв”Җ PROBLEMS:
в”Ӯ   в”ңв”Җв”Җ Activity Selection / Meeting Rooms
в”Ӯ   в”ңв”Җв”Җ Jump Game (LC 55)
в”Ӯ   в”ңв”Җв”Җ Jump Game II (LC 45)
в”Ӯ   в”ңв”Җв”Җ Gas Station (LC 134)
в”Ӯ   в”ңв”Җв”Җ Candy (LC 135)
в”Ӯ   в”ңв”Җв”Җ Task Scheduler (LC 621)
в”Ӯ   в”ңв”Җв”Җ Hand of Straights (LC 846)
в”Ӯ   в”ңв”Җв”Җ Merge Triplets (LC 1899)
в”Ӯ   в”ңв”Җв”Җ Partition Labels (LC 763) вӯҗ
в”Ӯ   в”ңв”Җв”Җ Valid Parenthesis String (LC 678)
в”Ӯ   в”ңв”Җв”Җ Non-overlapping Intervals (LC 435) вӯҗ
в”Ӯ   в””в”Җв”Җ Minimum Platforms (classic)
в”Ӯ
в””в”Җв”Җ GREEDY vs DP вҖ” HOW TO TELL:
    If greedy FAILS (can construct counter-example) вҶ’ use DP
    If greedy WORKS (can prove it) вҶ’ use Greedy (simpler)
```

### 7B. Bit Manipulation (Week 24-25)

```
CONCEPTS:
в”ңв”Җв”Җ AND (&), OR (|), XOR (^), NOT (~)
в”ңв”Җв”Җ Left shift (<<), Right shift (>>)
в”ңв”Җв”Җ Check if bit is set: (n >> i) & 1
в”ңв”Җв”Җ Set a bit: n | (1 << i)
в”ңв”Җв”Җ Clear a bit: n & ~(1 << i)
в”ңв”Җв”Җ Toggle a bit: n ^ (1 << i)
в”Ӯ
в”ңв”Җв”Җ KEY PROPERTIES:
в”Ӯ   в”ңв”Җв”Җ a ^ a = 0 (XOR with itself = 0)
в”Ӯ   в”ңв”Җв”Җ a ^ 0 = a (XOR with 0 = itself)
в”Ӯ   в”ңв”Җв”Җ n & (n-1) removes last set bit
в”Ӯ   в””в”Җв”Җ n & (-n) isolates last set bit
в”Ӯ
в”ңв”Җв”Җ PROBLEMS:
в”Ӯ   в”ңв”Җв”Җ Single Number (LC 136)
в”Ӯ   в”ңв”Җв”Җ Single Number II (LC 137)
в”Ӯ   в”ңв”Җв”Җ Single Number III (LC 260)
в”Ӯ   в”ңв”Җв”Җ Counting Bits (LC 338)
в”Ӯ   в”ңв”Җв”Җ Missing Number (LC 268)
в”Ӯ   в”ңв”Җв”Җ Reverse Bits (LC 190)
в”Ӯ   в”ңв”Җв”Җ Number of 1 Bits (LC 191)
в”Ӯ   в”ңв”Җв”Җ Sum of Two Integers without + (LC 371)
в”Ӯ   в””в”Җв”Җ Power of Two (LC 231)
```

```

# PART 6: COMMON MISTAKES TO AVOID

```
вқҢ MISTAKE 1: "I'll learn 3 languages for DSA"
вң… FIX: Pick ONE. Stick to it for 6 months. Period.

вқҢ MISTAKE 2: "Let me solve random LeetCode problems"
вң… FIX: Follow a structured sheet (Striver A2Z or NeetCode 150).
        Random solving = random results.

вқҢ MISTAKE 3: "I'll look at the solution after 5 minutes"
вң… FIX: Struggle for at least 20-30 minutes on Easy, 
        40-60 minutes on Medium. The struggle IS the learning.

вқҢ MISTAKE 4: "I solved it, moving on forever"
вң… FIX: Re-solve it after 3 days and 7 days. 
        If you can't solve it again, you didn't learn it.

вқҢ MISTAKE 5: "I'll skip Easy problems, they're too simple"
вң… FIX: Easy problems teach you patterns. 
        Medium problems combine patterns.
        Hard problems chain multiple patterns.
        You CANNOT skip the foundation.

вқҢ MISTAKE 6: "I'll just memorize solutions"
вң… FIX: Understand the PATTERN, not the solution.
        Two Sum teaches "complement lookup with HashMap."
        If you understand the pattern, you can solve 20 variants.
        If you memorize the code, you can solve only that 1 problem.

вқҢ MISTAKE 7: "DSA is enough for placement"
вң… FIX: You also need:
        вҶ’ 1-2 solid projects on resume
        вҶ’ Basic System Design knowledge
        вҶ’ OS, DBMS, CN fundamentals (CS core subjects)
        вҶ’ Communication skills for behavioral rounds

вқҢ MISTAKE 8: "I need to solve 1000 problems"
вң… FIX: 300 QUALITY problems with deep understanding
        > 1000 problems solved by copy-pasting.
        
вқҢ MISTAKE 9: "I'll start competitive programming to learn DSA"
вң… FIX: CP and interview DSA are DIFFERENT skills.
        CP: Speed + tricks + obscure algorithms
        Interview DSA: Patterns + communication + clean code
        Focus on interview DSA first. CP is optional.

вқҢ MISTAKE 10: "I'll start with DP because it's the hardest"
вң… FIX: DP requires recursion. Recursion requires arrays/strings.
        FOLLOW THE ORDER. There are no shortcuts.
```
